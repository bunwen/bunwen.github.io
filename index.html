关系大小调

一、级数

下面是一个C自然大调，它的每一个音都有一个级数

分别为 Ⅰ(一级)、Ⅱ(二级)、Ⅲ(三级)、Ⅳ(四级)、Ⅴ(五级)、Ⅵ(六级)、Ⅶ(七级)。

每一个调都有它的级数、每一个调的调內音的第一个音是它的Ⅰ级、第二个音是它的Ⅱ级，以此类推。

##二、属性
在一个调中，每一级都有它独特的属性

在一个调中第一级的属性叫做主、第二级叫做上主、第三级叫做中、第四级叫做下属、第五级叫做属、第六级叫做下中、第七级叫做导。

平常我们经常听到其他编曲大佬说什么主和弦、下属和弦、属和弦，则是以Ⅰ级、Ⅳ级、Ⅴ级去构建的和弦

以下图为例 在C自然大调中 主和弦是 1(do) 3(mi) 5(so)、下属和弦是 4(fa) 6(la) 1(do)、属和弦是 5(so) 7(si) 2(re)


而 在D自然大调中 主和弦是 2(re) 4#(fa) 6(la)、下属和弦是 5(so) 7(si) 2(re)、属和弦是 6(la) 1#(do) 3(mi)

##三、大小调
###3.1 大调
我们经常说一首歌是什么C大调的，G大调的，那么什么是大调呢？

大调就是它们每个调內音之间的的关系是 全全半全全全半 的音阶

比如C大调的调內音中 1(do) 2(re) 3(mi) 4(fa) 5(so) 6(la) 7(si)

1和2之间的关系是一个全音关系、2和3之间的关系是一个全音关系、3和4之间呢，是一个半音关系。以此类推啊，那么C大调的调內音之间的关系则分别全全半全全全半的关系。注意最后一个7他和下一个1的关系也是一个半音关系。

像这样音和音之间的关系是全全半全全全半的调我们就称他为大调

###3.2 小调
有大调那么自然就有小调，那么小调究竟是什么呢？

小调就是它们每个调內音之间的的关系是 全半全全半全全 的音阶

比如 a自然小调
a自然小调 的调內音分别是 6(la) 7(si) 1(do) 2(re) 3(mi) 4(fa) 5(so)
它们每个音之间的关系就满足 全半全全半全全 的规则

而像C自然大调和a自然小调这种音符全部相同但是顺序不一样的的调我们称他为：
关系大小调

需要注意的是，大调的书写方式是大写字母，而小调则是小写字母。
比如a自然小调我们就得写成 a自然小调 不能写成 A自然小调

总结一下：
这堂课我们学习了什么是级数，并且知道了每一级的属性是什么
我们还学习了什么是大调什么是小调，什么是关系大小调
在上一堂课中我们知道了在五线谱中不同大调的谱号。那么现在我们同样知道了他们的关系小调的谱号

练习：
1.写出所有大调。
2.写出它们的关系小调。
3.写出所有调的所有属性和弦

欢迎使用 MWeb

MWeb 是专业的 Markdown 写作、记笔记、静态博客生成软件，目前已支持 Mac，iPad 和 iPhone。MWeb 有以下特色：

软件本身：

  • 使用原生的 macOS 技术打造，追求与系统的完美结合。
  • 原则上，首先是追求界面简洁和高性能，然后才是强大易用，功能全面。

Markdown 语法：

  • 使用 GitHub Flavored Markdown 语法，简称 GFM 语法。
  • 支持表格、TOC、LaTeX、代码块、任务列表、脚注等。
  • 画图库支持 mermaid, viz, ECharts, PlantUML, Sequence, Flow。

Markdown 辅助：

  • 支持截图并粘贴、复制并粘贴、拖拽等方式插入图片并直接显示在编辑器内。
  • 在兼容 Markdown 语法的情况下支持设置图片宽度。
  • 好用的表格插入和 LaTeX 书写辅助。

Markdown 输出：

  • 支持导出为图片、HTML、Epub、PDF、RTF、Docx。
  • 支持发布到 WordPress、支持 Metaweblog API 协议的服务、Wordpress.com、印象笔记（Evernote）、Blogger、Medium、Tumblr、语雀、Ghost、少数派 Matrix。
  • 图片上传服务（图床）支持 Imgur、七牛云、又拍云、腾讯云 COS、阿里云 OSS、SM.MS和自定义的图床服务。

Markdown 笔记：

  • 强大的文档库支持分类树和标签管理文档，文档可归类于多个分类，可以把分类整个导出为 Epub、PDF 和生成静态网站。非常合适用于笔记、个人知识收集、管理和输出。
  • 快速笔记：随意增加笔记及图片等素材，支持以天为单位把增加的素材组合在一个文档内，方便整理及记录历史收集情况。
  • 快速搜索：目前已支持全局快捷键调出搜索。

外部 Markdown 文档：

  • 外部 Markdown 文档使用外部模式管理。外部模式使用目录树的方式编辑和管理你的 Markdown 文档，还能很好的支持 GitBook、Jekyll、Hexo 等等编辑和图片插入。

MWeb 文档：

如果要更详细了解 MWeb，建议你一定要去看一下官网的帮助文档，网址为：https://zh.mweb.im/help.html。我们建议在使用文档库之前，一定要阅读一下 MWeb 文档库详细介绍 这篇文章，以便更好的使用文档库。
